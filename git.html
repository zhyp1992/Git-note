查看本地仓库的状态（简短模式，文件状态：A 新增，M 修改，D 删除，R 重命名，?? 未跟踪） git diff # 比较工作目录与暂存区中的文件差异 git diff HEAD/ # 比较工作目录与（上次/指定）commit的文件差异 git diff --cached/--staged # 比较暂存区和上次commit的文件差异 git diff --stat git log [--oneline] # 显示commit历史，空格键滚动，按q退出，--oneline简洁模式（显示在一行） git log -n git log --graph --decorate --oneline # 显示commit历史，还有一些有用的选项。--graph 绘制图形，--decorate 显示commit分支名和标签 git log --stat # 显示每个commit被更改的文件及增删行数 git log -p # 显示每个commit全部的差异 git log # 查找特定文件的历史 git log --author="" # 与其他命令参数可组合使用，是字符串或正则表达式 git log --grep="" git log .. # since，until两个参数可以是commitID、分支名、HEAD/HEAD^^、tag引用 git log --oneline master..some-feature # ..句法，显示在some-feature分支而不在master分支的commit git show / # 查看某个commit信息， git show : # 查看某个commit中file文件信息 git reflog # 查看历史命令，reset --hard作死之后的后悔药 操作========================================================================================================================= //commit可以是commitID、分支名、HEAD/HEAD^^、tag等引用 //①使用git checkout 命令后，仓库将处于分离HEAD状态（不要在detached状态开发） //②执行reset --hard & clean -df推倒重来之后，工作目录和缓存区会返回到最后一次commit的状态，git status会认为这是一个干净的工作目录 //③--hard会改变工作目录，--soft不会改变暂存区和工作目录现有状态，--mixed与不加--mixed相同 git branch [-r/-a] # 列出本地仓库（远程仓库/本地+远程）所有分支名称 git branch -vv # 查看本地分支对应的远程分支 git branch -d/-D # 删除指定分支。未合并分支不能删除，需使用-D强制删除 git branch -m # 分支命名为branch git branch [--track] [] # 创建branch分支并追踪remote-branch（格式：origin/master） git branch --set-upstream # 设置已有branch分支追踪remote-branch git checkout git checkout -b # 基于exist-branch或某个commit创建新的分支并切换 *① git checkout # 将某个commit加载到工作目录，所有做改动不会被保存到仓库，git checkout master 切回之前状态 git checkout # 将某个commit中的file文件加载到工作目录并加入到缓存区，git checkout HEAD 切回之前状态 git checkout HEAD/master # 放弃对某个文件的修改 git checkout HEAD/master . # 放弃当前目录下的修改 git checkout # 将暂存区的文件恢复到工作目录中，暂存区文件依然存在 git stash save '' # 暂时保存变化的文件，可临时切换到其他分支工作，稍后切回使用git pop恢复 git stash [list/clear]/[pop/drop stash@''] git reset [/] # 从缓存区移除（全部/指定）文件，但不改变工作目录 *② git reset --hard # 清除所有未提交的更改（从缓存区移除文件，并将工作目录重置到更改前的状态） *② git clean -df [] # 移除（path文件夹下）未跟踪的文件及文件夹 *③ git reset [--mixed] # 撤销某次commit到缓存区，但不改变工作目录 git reset --hard # 不要强制用于公共仓库 git revert git merge --no-ff [master] # 将branch合并到当前所在分支，解决冲突后，add、commit完成合并。合并后可以删除branch，--no-ff不使用快速向前合并 git merge --squash # 合并分支时把分支中多余的commit合并 git rebase [-i] master # 将当前所在分支并入branch，与merge合并方式不同，不要用于已经推送到公共仓库的commit git rebase -i commit # 编辑、合并多个commit git rebase --continue # 解决冲突后在git add之后使用 git rebase --abort # 出错后，终止rebase git cherry-pick # 把某个commit合并到当前所在分支 远程========================================================================================================================= git tag git tag [-a] '' [-m ''] [] # 为（当前/指定）commit添加（带有描述信息的）标签 git tag -d '' git remote -v # 列出和其他仓库之间的远程连接，并显示连接的URL git remote add # 为远程连接添加别名，remote常用名origin git remote remove git remote rename git clone [-b ] [] # 克隆项目（的指定远程分支）到（目标）文件夹，repo为文件目录，HTTP或SSH链接，完成后自动设置origin git fetch [:] # 把远程仓库的指定分支更新到本地仓库（指定分支），推荐fetch到temp分支后自主选择合并 git pull [:] # git pull = git fetch + git merge（fetch获取远程分支HEAD，merge合并到当前分支） git pull --rebase # 获取远程仓库与本地合并，合并方式rebase不同于merge git push [-u] /[:] # 把本地仓库（指定分支或commit）推送到远程仓库指定分支，-u记忆后可省略参数直接使用git push git push --all/ # 推送（指定/所有）分支 git push --force # 强制推送，慎用 git push --tags/ # 提交（所有/指定）tag git push --delete # 删除远程tag，git push :refs/tags/ git push --delete # 删除远程分支，git push :refs/heads/ git fetch/pull -p # 如果远程的分支被删掉，可使用此命令删除本地对应分支 Git为了防止中央仓库的历史被覆盖，会拒绝所有导致非快速向前合并的推送请求。 所以，如果远程历史和你本地历史已经分叉，你需要将远程分支pull下来，在本地合并后再尝试推送。 fetch + merge同步本地和远程仓库： git fetch origin git checkout master git log --oneline master..origin/master git log origin/master git merge origin/master fetch + rebase同步本地和远程仓库：（推荐） git checkout master git fetch origin master git rebase -i origin/master # Squash commits, fix up commit messages etc. git push origin master pull + rebase同步本地和远程仓库： git checkout master git pull --rebase origin
